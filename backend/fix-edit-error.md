# Исправление: "Не удалось сохранить тип встречи" при редактировании

## Проблема

При попытке отредактировать тип встречи фронтенд показывал ошибку "Не удалось сохранить тип встречи".

## Причина

В методе `Patch` репозитория `EventTypeRepository` была **критическая ошибка с порядком аргументов SQL запроса**:

```go
// БЫЛО (НЕПРАВИЛЬНО):
args = append(args, id)  // ID добавлялся в args

query := fmt.Sprintf("... WHERE id = $%d ...", argIdx+1)

err = db.Pool.QueryRow(ctx, query, args...).Scan(...)
```

**Проблема**: ID добавлялся в `args` **ДО** формирования запроса, но запрос уже использовал `argIdx+1` как индекс. Это приводило к тому, что в запросе `$N` указывал на неправильное значение, и SQL запрос завершался ошибкой.

## Решение

Исправлен порядок добавления `id` в аргументы:

```go
// СТАЛО (ПРАВИЛЬНО):
query := fmt.Sprintf("... WHERE id = $%d ...", argIdx+1)

args = append(args, id)  // ID добавляется ПОСЛЕ формирования запроса

err = db.Pool.QueryRow(ctx, query, args...).Scan(...)
```

Теперь индекс `$N` в запросе корректно ссылается на последний аргумент (ID).

## Как проверить

### Вариант 1: Через фронтенд

```bash
# 1. Пересобрать и запустить
docker-compose down -v
docker-compose up -d

# 2. Открыть фронтенд http://localhost:80
# 3. Создать тип события
# 4. Кликнуть редактирование (иконка карандаша)
# 5. Изменить поля и нажать "Обновить"
# ✅ Ожидается: Успешное обновление
```

### Вариант 2: Через curl

```bash
# 1. Создать тип события
CREATE_RESPONSE=$(curl -s -X POST http://localhost:8080/api/event-types \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Тест",
    "description": "Тестовый тип",
    "durationMinutes": 30
  }')

EVENT_ID=$(echo "$CREATE_RESPONSE" | grep -o '"id":"[^"]*"' | cut -d'"' -f4)

# 2. Обновить тип события
curl -X PATCH http://localhost:8080/api/event-types/$EVENT_ID \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Тест (обновлено)",
    "description": "Обновлённое описание",
    "durationMinutes": 45
  }'

# ✅ Ожидается: HTTP 200 с обновлёнными данными
```

## Примечание

Эта ошибка была причиной того, что **любой PATCH запрос** завершался ошибкой базы данных, так как PostgreSQL получал неправильные параметры запроса. Фронтенд ловил эту ошибку и показывал generic сообщение "Не удалось сохранить тип встречи".
